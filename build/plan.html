Karma — la capa de confianza de la economía de agentes

Una economía donde los agentes AI se contratan y se pagan por-resultado on-chain, y cada interacción construye su reputación: la buena baja sus costos y le abre trabajo; la mala le pega un SBT calavera irrevocable que lo excluye del mercado.

"El Equifax de la economía de agentes — pero Equifax no te puede tatuar una calavera. Nosotros sí, y on-chain es para siempre. No nos importa quién sos; nos importa qué hiciste."

ScoreRegistry.sol A escribe

mapping(uint256 agentId => Score) mapping(bytes32 inputHash => Artifact) recordPayment(agentId, amount, inputHash) → emit PaymentRecorded(...) // arista + cache lookup(inputHash) view → Artifact setScore(agentId, value, nonce, sig) // ECDSA vs signer autorizado // si value >= GOOD_THRESHOLD → sbt.mint(GoodPayer) markDefault(agentId) onlyOwner → sbt.mint(agentId, Skull)

ReputationSBT.sol A escribe

// EIP-5192 · fork OZ ERC-721 enum Tier { GoodPayer, Skull } _update(...) // guard: revert si (from != 0 && to != 0) locked() → true supportsInterface incluye 0xb45a3c0e mint(agentId, tier) // solo desde ScoreRegistry

El SBT calavera es el corazón: si lo sacás, el score pierde dientes. Se pitchea desde acá.

ERC-8004 (solo lectura): IdentityRegistry + ReputationRegistry ya desplegados en Monad. Karma lee identidad/labels de los agentes — no los escribe.

07 Módulos

Cada submódulo con su responsabilidad y track. Sin división por personas. Tracks: Contratos Backend Front · otra persona, se acopla después

ScoreRegistry Contratos

Score por agente + cache de artefactos por inputHash. recordPayment emite la arista; setScore con firma ECDSA; markDefault dispara la calavera.

ReputationSBT Contratos

SBT EIP-5192 no transferible. Tiers GoodPayer y Skull. Guard en _update. Solo el ScoreRegistry puede mintear.

Server x402 Backend

Endpoint protegido que devuelve 402 y, tras settle exitoso, dispara la TX recordPayment. Flag DEMO_SAFE para mockear settle.

Cascada de agentes Backend

Orquestador con @x402/fetch que contrata 2-3 proveedores en cascada (texto mock o Claude). Consulta lookup antes de pagar.

Oráculo firmante Backend

Fórmula fija score = min(100, jobs*10 + volUSDC/10), firma con viem y postea setScore.

Grafo en vivo Front · después

React Flow + watchContractEvent sobre WS. Aristas animadas, score + badge Tier por nodo. Lo hace otra persona consumiendo nuestros eventos.

Comparador de costo Front · después

Contador de tx/seg + "Monad vs Ethereum". Refuerza el porqué-Monad en el demo. Parte del front (otra persona).

ReverseAuction Contratos stretch

Pujas bajando precio, ponderadas por reputación; bid de un agente con calavera revierte. Solo si el loop core anda a las 15:00.

09 Demo (3 min) — el héroe es la calavera

  • 0:00–0:25 · Problema. "Los agentes ya trabajan y se pagan, pero no hay reputación ni consecuencia si fallan. Construimos la capa de confianza." Grafo con agentes (identidad ERC-8004).
  • 0:25–1:05 · Reputación en vivo. Orquestador contrata y paga a 2-3 agentes vía x402 → grafo crece, scores suben, buen-pagador mintea SBT.
  • 1:05–1:35 · Beat de eficiencia (memoria). Un agente pide algo que ya está en el grafo → paga regalía chica al productor original. "El sistema recuerda; nadie paga dos veces lo mismo."
  • 1:35–2:25 · EL SHOWSTOPPER. Un agente entrega basura / no paga → markDefault → SBT calavera irrevocable aparece en su nodo → su próximo bid es rechazado on-chain en pantalla. (Abrir su identidad en el explorer: la calavera pegada.)
  • 2:25–3:00 · Por qué Monad + cierre. "Recalcular reputación y mintear en cada ciclo solo es viable a gas ~0 y sub-segundo. Implementamos el bid-weighting-por-reputación que el paper de Agent Exchange dejó en teoría. El Equifax de los agentes, con calavera on-chain."

Cómo no se pisan

S0 congela las superficies compartidas (backend/lib/*, env, ABIs). Cada sesión tiene FILES I OWN disjuntos + branch propio. B y C se encuentran en la interfaz TS ReputationLayer, nunca en el mismo archivo.

Cómo no se bloquean

Mocks/fixtures: A→forge test, B→MockReputationLayer, C→anvil/testnet, Front→fixtures de eventos. Nadie espera al deploy.

Cómo se une

MERGE cambia una línea (mock → OnchainReputationLayer), carga addresses reales y corre el loop core verificable por explorer.
